# AGENTS.md

Короткие постоянные правила для агентов в `ai-editor`.

## Контекст

`ai-editor` — локальный desktop-редактор русского текста с autocomplete. Текущая цель разработки — MVP 1. Подробный scope: `docs/MVP_1_SCOPE.md`.

Планируемый стек: Tauri 2, React, TypeScript, CodeMirror 6, Vitest, ESLint, Prettier, npm. Ollama подключается через adapter позже и не является обязательной частью MVP 1.

## Обязательные правила

- Перед задачей использовать `docs/CONTEXT_BUDGET.md`: выбрать только релевантные документы и кратко зафиксировать контекст.
- Читать `docs/MVP_1_SCOPE.md` перед задачами MVP 1.
- UI не вызывает напрямую Ollama, LLM, HTTP API или системные команды.
- Цепочка autocomplete: `Editor UI` -> `AutocompleteService` -> `SuggestionProvider` -> конкретный provider.
- Не смешивать UI, prompt-building и inference.
- Fallback autocomplete обязателен, даже если в MVP 1 он простой.
- Не реализовывать MVP 2 в задачах MVP 1.
- Перед UI-задачами читать `docs/UI_DESIGN_GUIDE.md`; при изменении визуального стиля обновлять guide или объяснять, почему это не нужно.
- Код писать на английском, документацию — на русском.
- Не хранить секреты, токены и API-ключи в репозитории.
- Не добавлять зависимости без проверяемой причины.
- Изменения делать небольшими, связными и проверяемыми.
- Архитектурные решения менять только через ADR.
- MCP подключены только проектно через `.codex/config.toml`; не требовать глобальный `codex mcp add`, кроме как явную альтернативу вне этого проекта.
- Context7 использовать только когда нужна актуальная документация внешней библиотеки.
- Playwright MCP использовать только для UI/e2e-сценариев, скриншотов и проверки поведения интерфейса.
- MCP и subagents не использовать автоматически «на всякий случай»; если задачу можно решить по локальному коду и локальным docs, MCP не использовать.
- Финальный отчёт держать коротким: изменения, проверки, ограничения, риски и затронутые файлы.

## Проверки

После scaffold ожидаются команды:

```bash
npm install
npm run dev
npm run build
npm run typecheck
npm run lint
npm run test
npm run check
```

Перед завершением задачи запускай доступные проверки. Если команды ещё не существуют, не выдумывай результат: напиши, что проект ещё не scaffolded.
