# Обзор проекта

`ai-editor` — локальный desktop-редактор для русского и английского текста с подсказками во время набора.

Пользователь пишет текст, видит короткие подсказки рядом с курсором, выбирает их стрелками вверх/вниз и принимает через `Tab`. Подсказка может быть словом, фразой или коротким продолжением предложения.

## Для кого

Проект ориентирован на людей, которые часто пишут русские и смешанные русско-английские тексты и хотят лёгкий локальный инструмент без аккаунтов и облачной синхронизации: заметки, черновики, рабочие тексты, личные документы.

Редактор не должен превращаться в чат-бота, IDE, админку или облачный сервис.

## Почему локально

Локальная модель развития выбрана ради приватности и предсказуемости:

- пользовательский текст не уходит во внешний сервис;
- приложение может работать без аккаунта;
- интеграции можно развивать постепенно и изолированно;
- dictionary provider позволяет развивать autocomplete без локальной LLM.

## Почему Tauri

Tauri даёт лёгкую desktop-оболочку, React/TypeScript UI и возможность держать Rust-часть минимальной. Нативный слой нужен только там, где frontend не справляется сам.

## Inference

Локальный inference рассматривается как заменяемый backend для providers. Целевой production runtime — bundled llama.cpp sidecar под управлением приложения. UI не должен знать о конкретной модели, endpoint, model path или HTTP-слое.

## Не входит в постоянные правила

Постоянная документация не управляет планом фич. Scope каждой итерации задаётся отдельной задачей, acceptance criteria и ограничениями в промте или task spec.
