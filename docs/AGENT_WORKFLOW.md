# Workflow агентной разработки

Одна задача = один понятный вертикальный срез. Не смешивать scaffold, UI, inference, тесты и рефакторинг без явного требования.

## Процесс

1. Прочитать `AGENTS.md` и relevant docs.
2. Сверить scope с `docs/MVP_1_SCOPE.md`.
3. Сформулировать короткий план для нетривиальной задачи.
4. Сделать минимальные изменения.
5. Запустить доступные проверки.
6. Кратко отчитаться: изменения, проверки, ограничения.

После крупных изменений можно запускать reviewer/tester subagents. Это рекомендация, а не обязательный ритуал для каждой мелкой правки.

## Роли

- `architect` — проверяет границы слоёв и избыточную сложность, не пишет код без команды.
- `coder` — реализует конкретную задачу, пишет тесты и запускает проверки.
- `reviewer` — ищет actionable проблемы в diff: баги, нарушение архитектуры, лишние зависимости, отсутствие тестов.
- `tester` — запускает проверки и сверяет acceptance criteria, не переписывает код без команды.
- `documenter` — обновляет документацию и ADR, не меняет код приложения без команды.
