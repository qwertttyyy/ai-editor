# UI Design Guide

Визуальная цель MVP 1: лёгкий локальный редактор в современном Apple-like стиле. Интерфейс спокойный, чистый, с фокусом на тексте. Он не должен выглядеть как админка, IDE или чат-бот.

## Layout

MVP 1 состоит из трёх зон:

- верхняя панель: название приложения, источник подсказок, переключатель темы, минимум кнопок;
- центральная область: большая мягкая карточка редактора с комфортной шириной строки и заметными внутренними отступами;
- нижняя status-панель: источник подсказок, состояние autocomplete, количество символов или слов, язык `RU`.

CodeMirror используется как основа редактора, но визуально не должен выглядеть как code editor: без плотной IDE-компоновки, лишних линий и визуального шума.

## Autocomplete popup

Popup появляется рядом с курсором как компактная floating-card:

- показывает 3-7 подсказок;
- выбранный пункт мягко подсвечен;
- тип подсказки можно показывать маленьким muted label;
- стрелки вверх/вниз меняют выбор;
- `Tab` принимает подсказку;
- `Esc` закрывает popup;
- popup не перекрывает большую часть текста.

## Темы

Нужны светлая и тёмная темы. Цвета должны быть нейтральными, мягкими и не кислотными.

Ориентиры:

- светлая тема: почти белый фон, белые поверхности, тонкие серые границы, тёмный основной текст, синий accent;
- тёмная тема: мягкий тёмно-серый фон, не абсолютный чёрный, светлый основной текст, приглушённый secondary text, синий accent;
- selection и focus state заметны, но не агрессивны;
- тени лёгкие, без тяжёлого material-like эффекта.

## Типографика и размеры

- Использовать системные шрифты, без внешних шрифтов в MVP 1.
- Текст редактора крупнее UI-текста: ориентир 17-19 px и line-height около 1.6.
- UI-текст компактный: ориентир 12-14 px.
- Отступы брать из шкалы `4, 8, 12, 16, 24, 32, 48 px`.
- Скругления мягкие: небольшие для кнопок и popup, крупнее для карточки редактора.

## Состояния

UI должен спокойно отображать:

- idle;
- autocomplete loading;
- autocomplete available;
- autocomplete empty;
- provider error;
- focus state.

Ollama-specific состояния добавляются только в задачах MVP 2.

## Запрещено

- кислотные цвета и яркие градиенты;
- перегруженные панели;
- плотная IDE-like компоновка;
- случайные размеры отступов;
- несколько разных шрифтов;
- большие навязчивые popup-окна;
- прямое копирование внешнего продукта один-в-один.

## Acceptance criteria

- Интерфейс визуально единый в светлой и тёмной теме.
- Редактор находится в центре внимания.
- Autocomplete popup читаемый и компактный.
- Status-панель не отвлекает.
- Интерфейс можно показать пользователю без объяснений.
