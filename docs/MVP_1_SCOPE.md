# Scope MVP 1

MVP 1 — минимальный рабочий редактор с mock autocomplete и архитектурой, готовой к замене источника подсказок. Всё, что не требуется для этого вертикального среза, относится к MVP 2+.

## Входит в MVP 1

- Tauri app запускается.
- React UI отображается.
- CodeMirror 6 подключён как редактор.
- Есть верхняя панель, центральная область редактора и нижняя status-панель.
- Есть светлая и тёмная тема.
- Есть mock autocomplete на 3-7 подсказок.
- Стрелки вверх/вниз выбирают подсказку.
- `Tab` вставляет выбранную подсказку.
- `Esc` закрывает popup.
- Есть `AutocompleteService`, `SuggestionProvider` и fallback-логика до mock-подсказок.
- Чистая autocomplete-логика покрыта unit-тестами.

## Можно добавить, если не раздувает scope

- Простой `DictionarySuggestionProvider`.
- Тип подсказки: слово, фраза, предложение.
- Счётчик символов или слов.
- Небольшие состояния `ready`, `loading`, `empty`, `error`.

## Не входит в MVP 1

- Реальный `OllamaSuggestionProvider` и health-check Ollama.
- Скачивание моделей.
- GPU/CPU detection.
- Выбор модели и сложные настройки inference.
- Полноценный менеджер файлов.
- Аккаунты и облачная синхронизация.
- llama.cpp.
- Система плагинов.
- Production installer.
- Telemetry и аналитика.
- Прямые вызовы Ollama, HTTP API или системных команд из UI.

## Acceptance criteria

- Приложение после scaffold запускается локально и показывает редактор.
- UI соответствует `docs/UI_DESIGN_GUIDE.md`.
- Пользователь может вводить русский текст.
- Mock autocomplete показывает, выбирает и вставляет подсказки.
- Fallback возвращает mock-подсказки или пустой безопасный результат при ошибке provider.
- Unit-тесты проверяют фильтрацию, выбор, вставку и fallback подсказок.
- UI не зависит напрямую от Ollama, HTTP API или системных команд.
