# Безопасность и permissions

`ai-editor` работает с пользовательским текстом, поэтому приватность и осторожность важнее удобных shortcuts.

## Секреты

- Не хранить токены, API-ключи и приватные credentials в репозитории.
- Не коммитить `.env` файлы.
- Не добавлять реальные endpoint с секретами в документацию или тесты.

## Network

- Не делать внешние network calls, если задача явно этого не требует.
- Для задач про Ollama допустим только локальный `localhost` endpoint, если task scope явно требует такую интеграцию.
- Не добавлять telemetry и аналитику.

## Файловая система

- Не читать домашнюю директорию пользователя без явной задачи.
- Не сканировать произвольные директории.
- Не удалять файлы без явного указания.
- Перед удалением файлов агент должен перечислить, что удаляет и зачем.

## Команды

- Не выполнять destructive-команды без явного указания.
- Не запускать неизвестные install scripts без причины.
- Не устанавливать лишние зависимости. Необходимые системные и проектные зависимости для scaffold, запуска, сборки или тестов можно устанавливать, если причина проверяема и отражена в отчёте.

## Работа Codex

Full-access Codex mode используется пользователем осознанно, но агент всё равно обязан минимизировать риск:

- менять только файлы в scope задачи;
- проверять diff перед отчётом;
- не трогать пользовательские локальные файлы без причины;
- не выдумывать результаты команд;
- явно сообщать, если проверка не была запущена.
